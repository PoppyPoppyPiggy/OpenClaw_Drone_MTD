Phase State Machine

Shows how many packets the agent classified into each attack phase. The OpenClaw agent uses a command-type state machine — not packet counting — to detect RECON→EXPLOIT→PERSIST→EXFIL transitions. COMMAND_LONG triggers EXPLOIT, PARAM_SET triggers PERSIST, LOG_REQUEST triggers EXFIL.

Phase Transition Log

Each row is a phase transition event — the exact command that caused the agent to reclassify the attacker's intent. This drives which response strategy the agent uses (rich telemetry for RECON, fake tokens for EXPLOIT, operator simulation for PERSIST, fake logs for EXFIL).

Tool Fingerprinting

The agent classifies attacker tools using inter-arrival timing + command sequence patterns (not string signatures — MAVLink is binary). nmap: burst <100ms + 5+ msg types. MAVProxy: HEARTBEAT at 1Hz. DroneKit: SET_MODE+DATA_STREAM. Metasploit: exploit commands + fast timing.

Autonomous Decision Timeline

The agent runs 5 async tasks that fire without external commands: proactive STATUSTEXT every 45s, sysid rotation every 120s, param cycling every 45s, mission refresh every 60s, and random proactive actions (flight simulation, ghost port hints, fake reboots). Each decision is logged for audit.

Response Coverage Matrix

Shows which (msg_type × phase) combinations produce a response vs return None. Green = agent generated contextual response. Red = fell through to MavlinkResponseGenerator fallback. After P1 fixes: all 5 test events return >0 bytes.

Bayesian Belief Trajectory

Tracks P(real_drone | observations) per attacker. Starts at prior=0.70. Each observation applies a likelihood ratio: BREADCRUMB_USE ×2.5 (attacker believes it's real), SCAN_DETECTED ×0.5 (suspicious), EVASION ×0.3 (suspects honeypot). The agent triggers MTD when P(real) drops below 0.5.

Packet Protocol Distribution

Breakdown of all captured packets by protocol layer. MAVLink/UDP is the primary attack surface (L0-L1). HTTP appears at L2 (API enumeration). WebSocket at L3 (CVE-2026-25253 exploit). TCP/Ghost at L4 (service chaining). Each protocol maps to different ATT&CK for ICS TTPs.

Per-Level Success Rate

Shows what percentage of packets at each attacker level received a valid response from the honey drone. L0 (random bytes) gets ~0% — expected. L1 (valid MAVLink) gets ~96%. L2 (HTTP) gets ~52%. L3 (WebSocket) gets ~80%. L4 (multi-service) gets ~90%. Higher = better deception.

DeceptionScore Waterfall

DS = w₁·(time_on_decoys) + w₂·(breach_prevention) + w₃·(avg_confusion) + w₄·(breadcrumb_follow) + w₅·(ghost_hit)
Each bar shows the raw metric × its weight = contribution to the final score. The cumulative green line builds up to DS=0.706. Weights: 0.30, 0.25, 0.20, 0.15, 0.10 (sum=1.0).

DS Baseline Comparison

Compares MIRAGE-UAS against two baselines: No Defense (attacker hits real drone directly, DS≈0.05) and Static Honeypot (fixed responses, no adaptation, DS≈0.20). MIRAGE-UAS achieves DS=0.706 — a 3.5× improvement over static honeypots through agentic phase-adaptive responses, breadcrumb planting, and ghost services.

Breadcrumb Flow Funnel

Tracks fake credentials through the deception pipeline: Planted (embedded in HTTP/WS responses) → Discovered (attacker parses response JSON) → Followed (attacker visits lure endpoints /config, /lure, /upload) → Extracted (attacker uses fake signing_key or api_token). Follow rate capped at 1.0.

MTD Action Latency

Execution time for each Docker SDK MTD action. PORT_ROTATE (iptables DNAT + mavlink-router HUP): 120ms avg. IP_SHUFFLE (Docker network disconnect/reconnect): 450ms. KEY_ROTATE (urandom + supervisor restart): 180ms. SERVICE_MIGRATE (full container replacement): 3200ms. All have 100% success rate.

MTD Cost Budget (Eq.17)

C_mtd(a) = κ · αᵢ · base_cost_i where κ=0.08 (grid search), αᵢ from 7D weight vector, base_cost from empirical measurement. Budget per trigger = urgency × 0.5. For L3 attacker (urgency=0.75), budget=0.375. The agent selects actions that fit within budget, with minimum 1 action guarantee.
